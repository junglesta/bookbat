# library-data-flow

Rules for editing library book data. Read this **before** opening, pasting into, or running anything that touches a `library.json` file.

## The three source files

| File | Role | Committed? |
|---|---|---|
| `data/library.json` | **Production seed** тАФ small curated set, the data that ships to production (BOOK BAT). The default source for `pnpm data:sync`. | yes |
| `data/library.full.json` | **Full dev dataset** тАФ the user's personal library, local-only. | no (gitignored) |
| `data/library.dummy.json` | 3-book minimal dataset used for tests/CI. | yes |

**Only these three files are ever hand-edited.** Everything under `apps/` is generated.

## Destinations (do not edit)

- `apps/bookbat/src/public/library.json`
- `apps/baobab/src/data/library.json`

Both are **gitignored** and regenerated by `pnpm data:sync` (or its variants). Anything pasted into them gets overwritten by the next sync. If a user reports pasting books "into the library JSON" and the books vanished, they almost certainly pasted into one of these destinations тАФ point them at `data/library.json` (seed) or `data/library.full.json` (full dev set) instead.

## Sync commands

| Script | Source | Use when |
|---|---|---|
| `pnpm data:sync` | `data/library.json` (seed) | Default. What the BOOK BAT production build uses. Use locally to preview the production seed. |
| `pnpm data:sync:full` | `data/library.full.json` (221+) | Local development with the full library. |
| `pnpm data:sync:dummy` | `data/library.dummy.json` (3) | Minimal dataset, e.g. for tests. |

## Dev / build commands

Default scripts use the **seed** (matches production):

- `pnpm dev:bookbat`, `pnpm dev:baobab`
- `pnpm build:bookbat`, `pnpm build:baobab`

Full-dataset variants exist for local development only:

- `pnpm dev:bookbat:full`, `pnpm dev:baobab:full`

**BOOK BAT production always uses the seed** тАФ its Cloudflare Workers Builds runs `build:bookbat`, keeping the deployed editor curated. **BAOBAB is the exception**: its production deploy ships the *full* library via `pnpm build:baobab:full` (see Deployment below).

## Deployment (Cloudflare Workers)

- **BOOK BAT (`bat.junglestar.org`)**: git-connected, auto-deploys from `main` via Cloudflare Workers Builds. Uses the seed (`data/library.json`). Configured by `apps/bookbat/wrangler.jsonc`.
- **BAOBAB (`baobab.junglestar.org`)**: **not** git-connected on purpose. The live site needs the full local library (`data/library.full.json`) which is gitignored. Deploy manually with `pnpm deploy:baobab` тАФ that script syncs from the full source, builds with `build:baobab:full`, and runs `wrangler deploy -c apps/baobab/wrangler.jsonc`.

Never enable BAOBAB git auto-deploy without changing this design; otherwise the live site would silently revert to the small seed every push.

## What to do when the user pastes new books

1. Confirm which file. If they pasted into a destination (`apps/**/library.json`), tell them and offer to move it.
2. Decide the intent:
   - "I want this to ship on the live site" тЖТ `data/library.json`
   - "I want this only on my machine for dev" тЖТ `data/library.full.json`
3. Do **not** run `pnpm data:sync` or any `dev`/`build` command without confirming, in case the source state isn't yet the truth.

## Webapp runtime data

The webapp's actual user library lives in browser localStorage at runtime. The seeded JSON above is only the initial dataset for empty installs. Editing JSON does not affect a user's already-populated webapp; that data is local to their browser.
