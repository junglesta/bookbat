# library-data-flow

Rules for editing library book data. Read this **before** opening, pasting into, or running anything that touches a `library.json` file.

## The three source files

| File | Role | Committed? |
|---|---|---|
| `data/library.json` | **Production seed** — small curated set, the data that ships on Netlify. The default source for `pnpm data:sync`. | yes |
| `data/library.full.json` | **Full dev dataset** — the user's personal library, local-only. | no (gitignored) |
| `data/library.dummy.json` | 3-book minimal dataset used for tests/CI. | yes |

**Only these three files are ever hand-edited.** Everything under `apps/` is generated.

## Destinations (do not edit)

- `apps/bookbat/src/public/library.json`
- `apps/baobab/src/data/library.json`

Both are **gitignored** and regenerated by `pnpm data:sync` (or its variants). Anything pasted into them gets overwritten by the next sync. If a user reports pasting books "into the library JSON" and the books vanished, they almost certainly pasted into one of these destinations — point them at `data/library.json` (seed) or `data/library.full.json` (full dev set) instead.

## Sync commands

| Script | Source | Use when |
|---|---|---|
| `pnpm data:sync` | `data/library.json` (seed) | Default. What Netlify runs. Use locally to preview the production seed. |
| `pnpm data:sync:full` | `data/library.full.json` (221+) | Local development with the full library. |
| `pnpm data:sync:dummy` | `data/library.dummy.json` (3) | Minimal dataset, e.g. for tests. |

## Dev / build commands

Default scripts use the **seed** (matches production):

- `pnpm dev:bookbat`, `pnpm dev:baobab`
- `pnpm build:bookbat`, `pnpm build:baobab`

Full-dataset variants exist for local development only:

- `pnpm dev:bookbat:full`, `pnpm dev:baobab:full`

**There is no `build:*:full`.** Production builds always use the seed by design — that's how the deployed site stays curated.

## Netlify

- **BOOKBAT (bookfreedom)**: git-connected, auto-deploys from `main`. Uses the seed (`data/library.json`, 9 books). Configured by root `netlify.toml`.
- **BAOBAB (baobab-junglestar-org)**: **not** git-connected on purpose. The live site needs the full local library (`data/library.full.json`, 221+ books) which is gitignored. Deploy manually with `pnpm deploy:baobab` — that script syncs from the full source, builds, and runs `netlify deploy --prod` against the BAOBAB site ID directly.

Never enable BAOBAB git auto-deploy without changing this design; otherwise the live site would silently revert to the 9-book seed every push.

## What to do when the user pastes new books

1. Confirm which file. If they pasted into a destination (`apps/**/library.json`), tell them and offer to move it.
2. Decide the intent:
   - "I want this to ship on the live site" → `data/library.json`
   - "I want this only on my machine for dev" → `data/library.full.json`
3. Do **not** run `pnpm data:sync` or any `dev`/`build` command without confirming, in case the source state isn't yet the truth.

## Webapp runtime data

The webapp's actual user library lives in browser localStorage at runtime. The seeded JSON above is only the initial dataset for empty installs. Editing JSON does not affect a user's already-populated webapp; that data is local to their browser.
